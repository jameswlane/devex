# Environment Configuration

The `environment.yaml` file manages development environment essentials: programming languages through mise, font installations, and shell configurations. This configuration ensures consistent tooling across development machines.

## File Location

```bash
# Default configuration
~/.local/share/devex/config/environment.yaml

# User override (higher priority)
~/.devex/environment.yaml
```

## File Structure

```yaml
programming_languages:
  - name: string          # Language/runtime name
    description: string   # Description of the language
    category: string      # "Programming Languages"
    install_method: string # Installation method (mise)
    install_command: string # Version specification
    # ... additional fields

fonts:
  - name: string          # Font name
    method: string        # Installation method
    url: string          # Download URL (if applicable)
    destination: string   # Installation path
    # ... font-specific options

shell: []                 # Shell configurations (extensible)
```

## Programming Languages

DevEx uses [mise](https://mise.jdx.dev/) for managing programming language versions. This provides consistent, project-aware version management across all languages.

### Language Definition Schema

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | Yes | Language identifier |
| `description` | `string` | Yes | Human-readable description |
| `category` | `string` | Yes | Always "Programming Languages" |
| `install_method` | `string` | Yes | Always "mise" |
| `install_command` | `string` | Yes | Version specification |
| `uninstall_command` | `string` | Yes | Uninstall identifier |
| `default` | `bool` | No | Install by default (false) |
| `dependencies` | `string[]` | No | Required dependencies |

## Font Management

DevEx manages programming fonts and system fonts through multiple installation methods.

### Font Schema

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | Yes | Font display name |
| `method` | `string` | Yes | Installation method |
| `url` | `string` | Conditional | Download URL (for url method) |
| `destination` | `string` | Conditional | Installation directory |
| `extract_path` | `string` | No | Path within archive to extract |

### Font Installation Methods

<Tabs items={['URL Download', 'Oh My Posh', 'Homebrew', 'Package Manager']}>
  <Tab value="URL Download">
    ```yaml
    fonts:
      - name: 'Cascadia Mono'
        method: 'url'
        url: 'https://github.com/ryanoasis/nerd-fonts/releases/latest/download/CascadiaMono.zip'
        destination: '~/.local/share/fonts'
        
      - name: 'iA Writer Mono'
        method: 'url'
        url: 'https://github.com/iaolo/iA-Fonts/archive/refs/heads/master.zip'
        extract_path: 'iA-Fonts-master/iA Writer Mono/Static'
        destination: '~/.local/share/fonts'
    ```
  </Tab>
  
  <Tab value="Oh My Posh">
    ```yaml
    fonts:
      - name: 'Meslo LG S'
        method: 'oh-my-posh'  # Uses oh-my-posh font installer
    ```
  </Tab>
  
  <Tab value="Homebrew">
    ```yaml
    fonts:
      - name: 'Fira Code'
        method: 'homebrew'    # macOS via Homebrew
    ```
  </Tab>
  
  <Tab value="Package Manager">
    ```yaml
    fonts:
      - name: 'Source Code Pro'
        method: 'apt'         # Linux via package manager
        package: 'fonts-source-code-pro'
        
      - name: 'Ubuntu Mono'
        method: 'dnf'         # Fedora via DNF
        package: 'ubuntu-family-fonts'
    ```
  </Tab>
</Tabs>

## Version Management with Mise

DevEx leverages mise for unified version management across all programming languages.

### Project-Level Version Control

```bash
# Create .mise.toml in project root
echo 'node = "18.17.0"' >> .mise.toml
echo 'python = "3.11.0"' >> .mise.toml

# Mise automatically switches versions when entering directory
cd my-project  # Automatically uses Node 18.17.0 and Python 3.11.0
```

### Global vs Project Versions

<Tabs items={['Global Configuration', 'Project Configuration', 'Version Switching']}>
  <Tab value="Global Configuration">
    ```bash
    # Set global defaults (used outside projects)
    mise use --global node@lts
    mise use --global python@3.11
    mise use --global go@latest
    
    # View global versions
    mise list
    ```
  </Tab>
  
  <Tab value="Project Configuration">
    ```toml
    # .mise.toml in project root
    [tools]
    node = "18.17.0"
    python = "3.11.0"
    go = "1.21.0"
    
    # Alternative: .tool-versions format
    # node 18.17.0
    # python 3.11.0
    # go 1.21.0
    ```
  </Tab>
  
  <Tab value="Version Switching">
    ```bash
    # Install and use specific version
    mise install python@3.12.0
    mise use python@3.12.0
    
    # Install all project versions
    mise install
    
    # Check current active versions
    mise current
    ```
  </Tab>
</Tabs>

## Best Practices

<Callout type="tip">
**Environment Configuration Best Practices:**

1. **Use LTS versions** - For production-focused environments
2. **Pin specific versions** - For team consistency
3. **Install Nerd Fonts** - Better terminal experience with symbols
4. **Leverage mise project configs** - Per-project version management
5. **Test font rendering** - Ensure fonts work in your terminal/editor
6. **Document team standards** - Share environment configs across team
</Callout>

## Validation and Testing

### Validate Environment Configuration

```bash
# Validate environment.yaml syntax
devex config validate environment

# Test programming language installation (dry-run)
devex install python --dry-run

# Verify mise is working
mise --version

# Check installed languages
mise list

# Test font installation
fc-list | grep -i "jetbrains"  # Linux
```

### Troubleshooting

<Tabs items={['Mise Issues', 'Font Problems', 'Version Conflicts']}>
  <Tab value="Mise Issues">
    ```bash
    # Mise not found
    curl https://mise.run | sh
    echo 'eval "$(mise activate bash)"' >> ~/.bashrc
    
    # Clear mise cache
    mise cache clear
    
    # Rebuild mise shims
    mise reshim
    
    # Check mise configuration
    mise doctor
    ```
  </Tab>
  
  <Tab value="Font Problems">
    ```bash
    # Linux: Refresh font cache
    fc-cache -fv
    
    # Check if fonts are installed
    fc-list | grep -i "font-name"
    
    # Verify font directory permissions
    ls -la ~/.local/share/fonts/
    
    # macOS: Font Book utility
    open /System/Applications/Font\ Book.app
    ```
  </Tab>
  
  <Tab value="Version Conflicts">
    ```bash
    # Check which versions are active
    mise current
    
    # See all available versions
    mise list-all python
    
    # Remove conflicting versions
    mise uninstall python@3.10.0
    
    # Reset to global version
    mise use --global python@3.11.0
    ```
  </Tab>
</Tabs>
