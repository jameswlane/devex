# System Configuration

The `system.yaml` file manages global system settings including Git configuration, SSH settings, terminal preferences, and global themes. These settings provide the foundation for a consistent development environment.

## File Location

```bash
# Default configuration
~/.local/share/devex/config/system.yaml

# User override (higher priority)
~/.devex/system.yaml
```

## File Structure

```yaml
git:
  aliases:               # Git command aliases
    co: 'checkout'
    br: 'branch'
    # ... more aliases
  settings:              # Git configuration settings
    pull:
      rebase: true
    # ... more settings
    
ssh: {}                  # SSH configuration (extensible)

terminal: {}             # Terminal configuration (extensible)

global_theme: 'Tokyo Night'  # Global theme preference
```

### Complete Git Configuration Example

```yaml
git:
  aliases:
    # Essential shortcuts
    co: 'checkout'
    br: 'branch'
    ci: 'commit'
    st: 'status'
    unstage: 'reset HEAD --'
    
    # Workflow aliases
    switch: 'checkout'
    new: 'checkout -b'
    save: 'commit -am'
    undo: 'reset --soft HEAD~1'
    
    # Logging and history
    lg: 'log --oneline --graph --decorate --all'
    hist: 'log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short'
    last: 'log -1 HEAD'
    
    # Branch operations
    branches: 'branch -a'
    remotes: 'remote -v'
    
  settings:
    # Workflow settings
    pull:
      rebase: true
    push:
      default: 'current'
      autoSetupRemote: true
    branch:
      autosetupmerge: 'always'
      autosetuprebase: 'always'
    fetch:
      prune: true
      
    # Merge and diff
    merge:
      conflictstyle: 'diff3'
    diff:
      colorMoved: 'default'
      
    # Core settings
    core:
      editor: 'code --wait'
      autocrlf: false  # Consistent line endings
      safecrlf: true
      
    # Color settings
    color:
      ui: 'auto'
      branch: 'auto'
      diff: 'auto'
      status: 'auto'
```

### Theme Integration

The global theme setting influences:

- **Neovim colorschemes**: Automatically set matching Neovim themes
- **Terminal themes**: Terminal color schemes
- **Desktop themes**: GNOME/KDE theme selection
- **VS Code themes**: Editor theme preferences
- **Application themes**: Consistent theming across development tools

```yaml
# Theme affects multiple configurations
global_theme: 'Tokyo Night'

# This influences:
# - ~/.config/nvim/colorscheme.lua
# - ~/.config/gnome/theme-settings
# - ~/.config/Code/User/settings.json (workbench.colorTheme)
# - ~/.config/alacritty/alacritty.yml (colors)
```

## Configuration Management

### Git Configuration Files

DevEx manages these Git configuration locations:

```bash
# Global Git config (managed by DevEx)
~/.gitconfig

# Repository-specific configs (user-managed)
project-dir/.git/config

# System-wide config (admin-managed)
/etc/gitconfig
```

### Configuration Precedence

1. **Repository-specific** (`.git/config`) - Highest priority
2. **User global** (`~/.gitconfig`) - DevEx managed
3. **System-wide** (`/etc/gitconfig`) - Lowest priority

### Applying System Configuration

```bash
# Apply all system configurations
devex system apply

# Apply only Git settings
devex system apply --git

# Preview changes (dry-run)
devex system apply --dry-run

# Check current Git configuration
git config --list --show-origin
```

## Troubleshooting

<Tabs items={['Git Issues', 'SSH Problems', 'Configuration Conflicts', 'Theme Issues']}>
  <Tab value="Git Issues">
    ```bash
    # Reset Git configuration
    git config --global --unset-all alias.co
    git config --global --unset pull.rebase
    
    # Check configuration conflicts
    git config --list --show-origin | grep pull.rebase
    
    # Debug Git operations
    GIT_TRACE=1 git pull
    
    # Repair repository
    git fsck
    git gc --prune=now
    ```
  </Tab>
  
  <Tab value="SSH Problems">
    ```bash
    # Debug SSH connection
    ssh -vT git@github.com
    
    # Check SSH agent
    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_ed25519
    
    # Test SSH config
    ssh -F ~/.ssh/config -T git@github.com
    
    # Reset SSH permissions
    chmod 700 ~/.ssh
    chmod 600 ~/.ssh/id_*
    chmod 644 ~/.ssh/id_*.pub
    ```
  </Tab>
  
  <Tab value="Configuration Conflicts">
    ```bash
    # Find configuration conflicts
    git config --list --show-origin | sort
    
    # Check for conflicting settings
    git config --get-all pull.rebase
    
    # Remove conflicting entries
    git config --global --unset-all setting.name
    
    # Backup current config
    cp ~/.gitconfig ~/.gitconfig.backup
    ```
  </Tab>
  
  <Tab value="Theme Issues">
    ```bash
    # Check theme application
    echo $DEVEX_THEME
    
    # Verify theme files exist
    ls ~/.local/share/devex/themes/
    
    # Test theme switching
    devex theme set tokyo-night
    
    # Reset to default theme
    devex theme reset
    ```
  </Tab>

## Best Practices

<Callout type="tip">
**System Configuration Best Practices:**

1. **Use meaningful aliases** - Short but descriptive Git shortcuts
2. **Enable rebase by default** - Cleaner Git history
3. **Sign commits** - For open source contributions
4. **Use SSH keys** - More secure than HTTPS
5. **Consistent themes** - Matching color schemes across tools
6. **Team standards** - Shared configurations for team consistency
7. **Regular updates** - Keep configurations current with new features
</Callout>
