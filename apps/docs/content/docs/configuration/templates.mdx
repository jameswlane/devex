# Template System

DevEx's template system allows you to create predefined configuration sets for different development scenarios, team setups, or project types. Templates make it easy to standardize environments and quickly bootstrap new development setups.

## Template Concepts

### What are Templates?

Templates are collections of configuration files that define a complete development environment setup. They can include:

- **Application selections**: Specific tools and software
- **Environment settings**: Programming languages, fonts, shell configs
- **Desktop configurations**: Themes, extensions, GUI applications
- **System settings**: Git config, SSH settings, terminal preferences

### Template Metadata

Each template includes a `template.yaml` file with metadata:

```yaml
# template.yaml
name: 'Web Development'
description: 'Full-stack web development environment'
version: '1.0.0'
author: 'DevEx Team'
tags: ['web', 'frontend', 'backend', 'javascript']

# Template configuration
includes:
  - applications.yaml
  - environment.yaml
  - desktop.yaml
  - system.yaml
  
# Variables for customization
variables:
  preferred_editor: 'vscode'  # or 'nvim', 'cursor'
  node_version: 'lts'         # or 'latest', '18.17.0'
  theme: 'tokyo-night'        # or 'gruvbox', 'catppuccin'
  
# Platform-specific overrides
platforms:
  linux:
    additional_packages: ['gnome-tweaks']
  macos:
    additional_packages: ['rectangle']
  windows:
    additional_packages: ['powertoys']
    
# Requirements
requirements:
  min_disk_space: '10GB'
  supported_platforms: ['linux', 'macos', 'windows']
  dependencies: ['git', 'curl']
```

### Data Science Template

Python-focused data science environment:

```yaml
# template.yaml
name: 'Data Science'
description: 'Python data science and ML environment'
version: '1.0.0'
tags: ['python', 'data-science', 'ml', 'jupyter']

variables:
  python_version: '3.11'
  jupyter_lab: true
  deep_learning: false

# applications.yaml
applications:
  development:
    - name: python
      install_command: 'python@${python_version}'
      default: true
    - name: jupyter
      default: '${jupyter_lab}'
    - name: vscode
      default: true
      
  optional:
    - name: r
      default: false
    - name: rstudio
      default: false
      
# environment.yaml  
programming_languages:
  - name: Python
    install_command: 'python@${python_version}'
    default: true
    post_install:
      - pip: 'pandas numpy matplotlib seaborn scikit-learn'
      - pip: 'jupyter jupyterlab'
      - pip: 'torch tensorflow'  # if deep_learning
```

### Creating a Mobile Development Template

```bash
# Create template directory
mkdir -p ~/.local/share/devex/templates/mobile-development

# Create template metadata
cat > ~/.local/share/devex/templates/mobile-development/template.yaml << 'EOF'
name: 'Mobile Development'
description: 'React Native and Flutter mobile development'
version: '1.0.0'
author: 'Mobile Team'
tags: ['mobile', 'react-native', 'flutter', 'ios', 'android']

variables:
  platform: 'react-native'  # or 'flutter', 'both'
  ios_development: false
  android_development: true
  
platforms:
  macos:
    ios_development: true  # Override for macOS
  windows:
    ios_development: false  # Force false on Windows
  linux:
    ios_development: false  # Force false on Linux
EOF
```

```yaml
# applications.yaml
applications:
  development:
    - name: node
      default: true
      condition: '${platform} == "react-native" || ${platform} == "both"'
    - name: flutter
      default: true
      condition: '${platform} == "flutter" || ${platform} == "both"'
    - name: android-studio
      default: '${android_development}'
    - name: xcode
      default: '${ios_development}'
      platforms: ['macos']
      
  tools:
    - name: scrcpy  # Android device mirroring
      default: '${android_development}'
    - name: flipper  # Mobile debugging
      default: true
      
# environment.yaml
programming_languages:
  - name: Node.js
    install_command: 'node@lts'
    default: true
    condition: '${platform} != "flutter"'
  - name: Dart
    install_command: 'dart@latest'
    default: true
    condition: '${platform} != "react-native"'
  - name: Java
    install_command: 'java@11'
    default: '${android_development}'
```

## Template Usage

### Applying Templates

```bash
# List available templates
devex template list

# Apply a template
devex template apply web-development

# Apply with custom variables
devex template apply web-development \
  --var node_version=18.17.0 \
  --var theme=gruvbox \
  --var preferred_editor=nvim

# Preview template changes (dry-run)
devex template apply data-science --dry-run

# Apply template to specific configuration
devex template apply team-standard --config-only applications
```

  </Tab>
  
  <Tab value="Boolean Variables">
    ```yaml
    # Template definition
    variables:
      include_docker: true
      install_databases: false
      
    # Usage in configuration
    applications:
      development:
        - name: docker
          default: '${include_docker}'
        - name: postgresql
          default: '${install_databases}'
          
    # CLI usage
    devex template apply web-dev \
      --var include_docker=false \
      --var install_databases=true
    ```
  </Tab>
  
  <Tab value="Conditional Logic">
    ```yaml

        - name: android-studio
          default: true
          condition: '${platform} == "android" || ${platform} == "both"'
          
    # Complex conditions
    - name: flutter
      default: true
      condition: '(${framework} == "flutter") && (${mobile_dev} == true)'
    ```

## Best Practices

<Callout type="tip">
**Template Best Practices:**

1. **Keep templates focused** - One template per use case
2. **Use meaningful variables** - Clear parameter names and defaults
3. **Document thoroughly** - Include usage examples and customization guides
4. **Version templates** - Track changes and maintain compatibility
5. **Test across platforms** - Ensure templates work on target systems
6. **Provide fallbacks** - Handle missing dependencies gracefully
7. **Regular updates** - Keep templates current with tool updates
</Callout>

### Template Naming Conventions

```bash
# Good template names
web-development        # Clear, descriptive
data-science-python   # Specific technology focus
team-standard-2024    # Versioned team configs
mobile-react-native   # Technology-specific

# Avoid
frontend               # Too generic
my-setup              # Not descriptive
template1             # Not meaningful
```

### Variable Design

```yaml
# Good variable design
variables:
  # Clear names with sensible defaults
  node_version: 'lts'           # Not 'nv' or 'version'
  include_testing_tools: true   # Not 'testing'
  database_type: 'postgresql'  # Not 'db'
  
  # Provide documentation
  theme: 'tokyo-night'  # Available: tokyo-night, gruvbox, catppuccin
```
