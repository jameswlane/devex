## How to Use These Examples

Each example provides:

### 1. Complete Configuration Files
Ready-to-use YAML files that you can copy to your `~/.devex/` directory:

```bash
# Copy an example configuration
curl -fsSL https://raw.githubusercontent.com/jameswlane/devex/main/examples/web-development.yaml > ~/.devex/web-dev.yaml

# Apply the configuration
devex config apply --file ~/.devex/web-dev.yaml
devex install
```

### 2. Step-by-Step Setup Instructions
Detailed walkthrough of the setup process, including:
- Prerequisites and system requirements
- Installation commands
- Configuration explanations
- Verification steps

### 3. Customization Options
Guidelines for adapting the examples to your specific needs:
- Adding or removing tools
- Modifying versions
- Team-specific adjustments

### 4. Best Practices
Proven approaches for:
- Team collaboration
- CI/CD integration
- Environment management
- Troubleshooting

## Common Patterns

### Multi-Environment Setup

Most examples follow this pattern for handling different environments:

```yaml
# Base configuration for all environments
applications:
  development:
    - name: git
      default: true
    - name: docker
      default: true

# Development-specific tools
development:
  applications:
    - name: docker-compose
      default: true
    - name: dev-tools
      default: true

# Production-specific configurations
production:
  applications:
    - name: monitoring-agent
      default: true
  security:
    enabled: true
```

### Team Collaboration Structure

Examples include team-friendly configurations:

```yaml
# Team base configuration
team:
  name: "frontend-team"
  tools:
    editor: "vscode"
    shell: "zsh"
    theme: "tokyo-night"

# Role-based additions
roles:
  senior-developer:
    additional_tools:
      - kubernetes-cli
      - terraform
  
  junior-developer:
    guided_setup: true
    documentation_tools:
      - devdocs
```

### Cross-Platform Compatibility

All examples include platform-specific configurations:

```yaml
applications:
  development:
    - name: node
      linux:
        install_method: mise
        install_command: node@lts
      macos:
        install_method: mise
        install_command: node@lts
      windows:
        install_method: mise
        install_command: node@lts
```

## Quick Start with Examples

### 1. Explore Available Examples

```bash
# List all available examples
devex examples list

# Show example details
devex examples show web-development

# Preview what would be installed
devex examples preview web-development
```

### 2. Apply an Example

```bash
# Apply example directly
devex examples apply web-development

# Or download and customize
devex examples download web-development --output ~/.devex/my-config.yaml
# Edit ~/.devex/my-config.yaml
devex config apply --file ~/.devex/my-config.yaml
```

### 3. Combine Multiple Examples

```bash
# Combine examples for full-stack development
devex examples apply web-development backend-development

# Merge specific categories
devex examples apply web-development --category frontend
devex examples apply backend-development --category databases
```

## Contributing Examples

Have a useful DevEx configuration? Contribute it to help others!

### Example Structure

```
examples/
├── example-name/
│   ├── README.md                 # Description and instructions
│   ├── devex.yaml              # Main configuration
│   ├── team/                   # Team variations
│   │   ├── frontend.yaml
│   │   └── backend.yaml
│   ├── environments/           # Environment-specific configs
│   │   ├── development.yaml
│   │   ├── staging.yaml
│   │   └── production.yaml
│   └── assets/                 # Supporting files
│       ├── vscode-settings.json
│       └── git-hooks/
```

### Contribution Guidelines

1. **Complete Documentation**: Include clear README with setup instructions
2. **Cross-Platform Support**: Test on Linux, macOS, and Windows
3. **Security Best Practices**: Follow security guidelines
4. **Real-World Testing**: Verify the configuration works in practice
5. **Team Feedback**: Get feedback from actual users

```bash
# Fork the repository
git clone https://github.com/yourusername/devex.git

# Create your example
mkdir -p examples/my-development-stack
cd examples/my-development-stack

# Create configuration files
# ... add your files ...

# Test the example
devex config apply --file devex.yaml --dry-run
devex examples test my-development-stack

# Submit pull request
git add .
git commit -m "Add my-development-stack example"
git push origin my-example-branch
```

<Callout type="tip">
**Example Best Practices:**

1. **Start Simple**: Begin with essential tools, add complexity gradually
2. **Document Decisions**: Explain why specific tools were chosen
3. **Include Alternatives**: Provide options for different preferences
4. **Test Thoroughly**: Verify on clean systems and different platforms
5. **Keep Updated**: Maintain examples as tools and versions evolve
6. **Real-World Focus**: Base examples on actual development needs
</Callout>
