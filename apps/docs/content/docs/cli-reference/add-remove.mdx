# devex add & remove

The `add` and `remove` commands allow you to dynamically modify your development environment by adding new applications or removing existing ones. These commands provide both interactive and non-interactive modes for managing your software installations.

## devex add

Add new applications to your development environment configuration and optionally install them immediately.

### Usage

```bash
devex add [application...] [flags]
```

### Options

| Flag | Short | Type | Default | Description |
|------|-------|------|---------|-------------|
| `--install` | `-i` | `bool` | `false` | Install immediately after adding to config |
| `--category` | `-c` | `string` | | Filter applications by category |
| `--search` | `-s` | `string` | | Search applications by name or description |
| `--interactive` | | `bool` | `true` | Use interactive selection mode |
| `--force` | `-f` | `bool` | `false` | Add even if already configured |
| `--dry-run` | `-n` | `bool` | `false` | Show what would be added without executing |

## devex remove

Remove applications from your development environment configuration with intelligent dependency checking.

### Usage

```bash
devex remove [application...] [flags]
```

### Options

| Flag | Short | Type | Default | Description |
|------|-------|------|---------|-------------|
| `--uninstall` | `-u` | `bool` | `false` | Uninstall from system after removing from config |
| `--force` | `-f` | `bool` | `false` | Remove even if other apps depend on it |
| `--backup` | `-b` | `bool` | `true` | Create backup before removal |
| `--interactive` | | `bool` | `true` | Use interactive selection mode |
| `--dry-run` | `-n` | `bool` | `false` | Show what would be removed without executing |

### Dependency Safety

<Callout type="warn">
DevEx automatically analyzes dependencies before removal to prevent breaking your environment. Applications with dependents are marked with warnings and require explicit confirmation.
</Callout>

### Dependency Management

DevEx analyzes three types of dependencies:

#### Direct Dependencies
Applications that explicitly require the target application:

```yaml
applications:
  development:
    - name: docker-compose
      dependencies:
        - docker  # Direct dependency
```

#### System Dependencies
Applications that rely on system-level integration:

```
kubectl → docker (for local Kubernetes clusters)
VS Code → git (for version control features)
npm packages → node (runtime requirement)
```

#### Optional Dependencies
Applications that enhance functionality but aren't required:

```
Postman → Chrome (for OAuth flows, optional)
VS Code → Python (for Python development, optional)
```

### Removal Strategies

<Tabs items={['Conservative', 'Aggressive', 'Selective', 'Bulk']}>
  <Tab value="Conservative">
    **Conservative Removal**: Only remove applications with no dependencies
    
    ```bash
    # Check what's safe to remove
    devex remove --dry-run --safe-only
    
    # Remove only applications without dependents
    devex remove slack postman chrome
    ```
  </Tab>
  
  <Tab value="Aggressive">
    **Aggressive Removal**: Remove everything, handling dependencies
    
    ```bash
    # Remove with cascade (removes dependents too)
    devex remove docker --cascade
    
    # Force remove core dependencies
    devex remove git --force --cascade
    ```
  </Tab>
  
  <Tab value="Selective">
    **Selective Removal**: Choose specific applications with dependency resolution
    
    ```bash
    # Interactive dependency resolution
    devex remove docker  # Will prompt about dependents
    
    # Remove and replace dependencies
    devex remove docker --replace-with=podman
    ```
  </Tab>
  
  <Tab value="Bulk">
    **Bulk Removal**: Remove entire categories
    
    ```bash
    # Remove all optional applications
    devex remove --category optional
    
    # Remove unused applications
    devex remove --unused
    
    # Remove applications not used in 30 days
    devex remove --unused --days=30
    ```
  </Tab>
</Tabs>

### Undo Support

Both commands support undo operations:

```bash
# View recent changes
devex config undo list

# Undo last add operation
devex config undo last

# Undo specific operation
devex config undo operation "add docker vscode"

# Undo with confirmation
devex config undo operation 3 --confirm
```

## Advanced Usage

### Custom Application Definitions

Add custom applications not in the default catalog:

```bash
# Add custom application with definition
devex add custom-tool --define \
  --name="Custom Tool" \
  --description="My custom development tool" \
  --install-method="curlpipe" \
  --install-command="https://example.com/install.sh"

# Add from custom configuration file
devex add --from-file ./my-custom-apps.yaml
```

### Bulk Operations

Perform operations on multiple applications:

```bash
# Add entire development stack
devex add @full-stack  # Uses predefined template

# Remove development tools except essentials
devex remove --category development --except git,curl

# Add applications matching pattern
devex add "*code*"  # Adds vscode, code-server, etc.

# Remove unused applications
devex remove --cleanup --unused
```

### Integration with Templates

Add/remove commands work with configuration templates:

```bash
# Add applications from template
devex template apply backend-team
devex add --from-template backend-team

# Remove applications not in template
devex remove --not-in-template backend-team

# Sync with team template (add missing, remove extra)
devex add --sync-template backend-team
```

  <Tab value="Dependency Problems">
