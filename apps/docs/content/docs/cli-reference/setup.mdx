# devex setup

The `setup` command provides an interactive, guided installation experience for configuring your development environment. It walks you through selecting programming languages, databases, and applications to create a customized setup tailored to your needs.

## Usage

```bash
devex setup [flags]
```

## Description

The setup command includes:

- **Programming language selection** (Node.js, Python, Go, Ruby, etc.)
- **Database installation** (PostgreSQL, MySQL, Redis)
- **Essential development tools** and terminal applications
- **Desktop applications** (if desktop environment detected)
- **Automatic dependency management** and ordering
- **Git configuration** setup
- **Theme and desktop environment** configuration

## Options

| Flag | Short | Type | Default | Description |
|------|-------|------|---------|-------------|
| `--non-interactive` | | `bool` | `false` | Run automated setup without user interaction |
| `--verbose` | `-v` | `bool` | `false` | Enable verbose output |
| `--dry-run` | `-n` | `bool` | `false` | Show what would be installed without executing |

## Examples

### Basic Setup

```bash
# Start interactive guided setup
devex setup
```

### Non-Interactive Setup

```bash
# Run with default selections
devex setup --non-interactive

# Non-interactive with verbose output
devex setup --non-interactive --verbose
```

### Preview Mode

```bash
# See what would be installed without doing it
devex setup --dry-run
```

## Configuration Integration

The setup command works with your configuration files in `~/.local/share/devex/config/`:

<Tabs items={['Default Behavior', 'Custom Configs', 'Team Configs']}>
  <Tab value="Default Behavior">
    Uses built-in default configurations for all selections. Perfect for getting started quickly.
    
    ```bash
    devex setup
    # Uses defaults from:
    # - applications.yaml
    # - environment.yaml  
    # - desktop.yaml
    # - system.yaml
    ```
  </Tab>
  
  <Tab value="Custom Configs">
    If you have custom configurations in `~/.devex/`, setup will use those instead.
    
    ```bash
    # Create custom config
    mkdir -p ~/.devex
    cp ~/.local/share/devex/config/applications.yaml ~/.devex/
    # Edit ~/.devex/applications.yaml
    
    devex setup  # Now uses your custom config
    ```
  </Tab>
  
  <Tab value="Team Configs">
    Use a team configuration template for consistent environments.
    
    ```bash
    # First get the team template
    devex template list
    devex template apply backend-team
    
    # Then run setup
    devex setup
    ```
  </Tab>
</Tabs>

## Error Handling

<Callout type="warn">
If setup encounters errors, it will:

1. **Continue with other installations** when possible
2. **Show a summary** of what succeeded and failed
3. **Provide specific error messages** for troubleshooting
4. **Suggest recovery commands** for common issues
</Callout>

### Common Issues

**Permission Errors:**
```bash
# Setup will prompt for sudo when needed
# If issues persist, check:
sudo chmod 755 /usr/local/bin
```

**Network Issues:**
```bash
# Retry with verbose output
devex setup --verbose

# Check network connectivity
curl -I https://github.com
```

**Package Manager Issues:**
```bash
# Update package lists first
sudo apt update  # Ubuntu/Debian
brew update      # macOS
```

## Advanced Options

### Custom Installation Order

Setup automatically determines the optimal installation order based on dependencies, but you can influence this:

```yaml
# In your applications.yaml
applications:
  development:
    - name: docker
      priority: 1  # Install first
    - name: node
      priority: 2  # Install after docker
```

### Platform-Specific Behavior

Setup adapts to your platform:

<Tabs items={['Linux', 'macOS', 'Windows']}>
  <Tab value="Linux">
    - Detects distribution (Ubuntu, Fedora, Arch, etc.)
    - Uses appropriate package manager (apt, dnf, pacman)
    - Configures desktop environment (GNOME, KDE, XFCE)
    - Sets up systemd services when needed
  </Tab>
  
  <Tab value="macOS">
    - Uses Homebrew as primary package manager
    - Configures macOS defaults and dock
    - Installs from Mac App Store when available
    - Sets up macOS-specific development tools
  </Tab>
  
  <Tab value="Windows">
    - Uses winget as primary package manager
    - Falls back to Chocolatey or Scoop
    - Configures Windows Terminal
    - Sets up WSL if requested
  </Tab>
</Tabs>

## Post-Setup

After setup completes:

<Steps>
  <Step>**Review Installation**: Check the summary for any failed installations</Step>
  <Step>**Shell Reload**: Restart your terminal or run `source ~/.zshrc`</Step>
  <Step>**Verify Tools**: Run `devex status` to confirm everything installed</Step>
  <Step>**Customize**: Use `devex config` to fine-tune your setup</Step>
</Steps>
