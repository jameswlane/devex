# devex config

The `config` command provides comprehensive tools for managing DevEx configuration files. It includes subcommands for viewing, editing, validating, and backing up your configuration files, as well as advanced features like team configuration management and environment-specific settings.

## Usage

```bash
devex config <subcommand> [flags]
```

### devex config edit

Open configuration files in your preferred editor.

```bash
devex config edit <file> [flags]
```

#### Arguments

- `file`: Configuration file to edit (`applications`, `environment`, `desktop`, `system`)

#### Options

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--editor` | `string` | `$EDITOR` | Editor to use (vim, nano, code, etc.) |
| `--backup` | `bool` | `true` | Create backup before editing |
| `--template` | `string` | | Start from a template |

#### Examples

```bash
# Edit applications configuration
devex config edit applications

# Use specific editor
devex config edit environment --editor code

# Edit with automatic backup
devex config edit desktop --backup

# Start from template
devex config edit applications --template backend-team
```

### devex config diff

Compare configuration files or show changes.

```bash
devex config diff <source> [target] [flags]
```

#### Options

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--unified` | `int` | `3` | Number of context lines |
| `--ignore-whitespace` | `bool` | `false` | Ignore whitespace differences |
| `--semantic` | `bool` | `false` | Show semantic differences only |

#### Examples

```bash
# Compare with backup
devex config diff applications backup-20240817.yaml

# Compare current with default
devex config diff applications default

# Show changes since last commit
devex config diff --since-commit

# Semantic comparison (ignores formatting)
devex config diff applications backup.yaml --semantic
```

### Environment-Specific Configuration

Handle different configurations for different environments (development, staging, production).

```bash
devex config environment <subcommand> [flags]
```

#### Environment Subcommands

```bash
# List available environments
devex config environment list

# Show current environment
devex config environment status

# Switch to different environment
devex config environment use production

# Create new environment configuration
devex config environment init staging --from development
```

#### Environment Example

<Steps>
  <Step>
    **Create Environment**
    ```bash
    devex config environment init staging
    ```
  </Step>
  
  <Step>
    **Customize for Environment**
    ```bash
    devex config edit applications --env staging
    # Remove development-only tools
    # Add staging-specific monitoring
    ```
  </Step>
  
  <Step>
    **Switch Environment**
    ```bash
    devex config environment use staging
    devex install  # Installs staging configuration
    ```

</Steps>

## Import and Export

### Export Configurations

```bash
devex config export [flags]
```

#### Export Options

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--output` | `string` | `devex-config.tar.gz` | Output file path |
| `--format` | `string` | `archive` | Export format (archive, directory, git) |
| `--include-backups` | `bool` | `false` | Include backup files |
| `--compress` | `bool` | `true` | Compress the export |

#### Export Examples

```bash
# Export complete configuration
devex config export

# Export to specific file
devex config export --output my-devex-setup.tar.gz

# Export as directory
devex config export --format directory --output ./my-config

# Export for team sharing (includes templates)
devex config export --team --include-templates
```

### Import Configurations

```bash
devex config import <source> [flags]
```

#### Import Options

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--merge` | `bool` | `false` | Merge with existing configuration |
| `--backup` | `bool` | `true` | Create backup before import |
| `--validate` | `bool` | `true` | Validate imported configuration |

#### Import Examples

```bash
# Import configuration package
devex config import devex-config.tar.gz

# Import and merge with existing
devex config import team-config.tar.gz --merge

# Import from directory
devex config import ./config-directory

# Import from Git repository
devex config import https://github.com/company/devex-configs.git
```

## Configuration Inheritance

DevEx supports configuration inheritance for complex scenarios:

```bash
devex config inheritance <subcommand> [flags]
```

### Inheritance Chain

```
User Override (~/.devex/)
        ↓
Team Configuration
        ↓
Environment Configuration
        ↓
Default Configuration
```

### Inheritance Commands

```bash
# Show inheritance chain
devex config inheritance show

# Show effective configuration (after inheritance)
devex config inheritance effective

# Debug inheritance issues
devex config inheritance debug
```

## Best Practices

<Callout type="tip">
**Configuration Management Best Practices:**

1. **Always backup** before major changes
2. **Use team configurations** for consistent environments
3. **Validate regularly** to catch issues early
4. **Version control** your custom configurations
5. **Document changes** with meaningful commit messages
</Callout>

<Steps>
  <Step>
    ### Backup Current Configuration
    ```bash
    devex config backup create --name "before-changes"
    ```
  </Step>
  
  <Step>
    ### Make Changes Safely
    ```bash
    devex config edit applications --backup
    ```
  </Step>
  
  <Step>
    ### Validate Changes
    ```bash
    devex config validate --strict
    ```
  </Step>
  
  <Step>
    ### Test with Dry Run
    ```bash
    devex install --dry-run
    ```
  </Step>
  
  <Step>
    ### Apply if Successful
    ```bash
    devex install
    ```
  </Step>
