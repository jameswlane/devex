# Global Flags & Completion

DevEx provides a comprehensive set of global flags, environment variables, and shell completion features that work across all commands to enhance your development experience.

## Global Flags

These flags are available for all DevEx commands and provide consistent behavior across the CLI.

### Core Global Flags

| Flag | Short | Type | Default | Description |
|------|-------|------|---------|-------------|
| `--verbose` | `-v` | `bool` | `false` | Enable verbose output with detailed information |
| `--dry-run` | `-n` | `bool` | `false` | Show what would be done without executing |
| `--help` | `-h` | `bool` | `false` | Show help information for the command |
| `--version` | | `bool` | `false` | Show DevEx version information |

### Configuration Flags

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--config-dir` | `string` | `~/.local/share/devex/config` | Override default configuration directory |
| `--user-config-dir` | `string` | `~/.devex` | Override user configuration directory |
| `--cache-dir` | `string` | `~/.cache/devex` | Override cache directory |
| `--no-backup` | `bool` | `false` | Disable automatic backups |

### Output Flags

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--output` | `string` | `auto` | Output format: table, json, yaml, csv |
| `--no-color` | `bool` | `false` | Disable colored output |
| `--quiet` | `bool` | `false` | Suppress non-essential output |
| `--debug` | `bool` | `false` | Enable debug logging |

## Environment Variables

DevEx recognizes several environment variables that control its behavior globally.

### Core Environment Variables

<Cards>
  <Card title="DEVEX_DEBUG" description="Enable debug logging (true/false)" />
  <Card title="DEVEX_CONFIG_DIR" description="Override default configuration directory" />
  <Card title="DEVEX_NO_COLOR" description="Disable colored output (true/false)" />
  <Card title="DEVEX_CACHE_DIR" description="Override cache directory location" />
</Cards>

### Configuration Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `DEVEX_CONFIG_DIR` | `~/.local/share/devex/config` | Default configuration directory |
| `DEVEX_USER_CONFIG_DIR` | `~/.devex` | User override configuration directory |
| `DEVEX_CACHE_DIR` | `~/.cache/devex` | Cache directory for downloads and metadata |
| `DEVEX_BACKUP_DIR` | `~/.devex/backups` | Directory for configuration backups |
| `DEVEX_LOG_DIR` | `~/.local/share/devex/logs` | Directory for log files |

### Behavior Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `DEVEX_DEBUG` | `false` | Enable debug logging and verbose output |
| `DEVEX_NO_COLOR` | `false` | Disable colored terminal output |
| `DEVEX_NO_BACKUP` | `false` | Disable automatic backup creation |
| `DEVEX_AUTO_YES` | `false` | Automatically answer "yes" to prompts |
| `DEVEX_TIMEOUT` | `300` | Command timeout in seconds |

### Platform-Specific Variables

| Variable | Platform | Description |
|----------|----------|-------------|
| `DEVEX_PACKAGE_MANAGER` | Linux | Force specific package manager (apt, dnf, pacman) |
| `DEVEX_USE_SUDO` | Linux | Control sudo usage (true/false/auto) |
| `DEVEX_HOMEBREW_PREFIX` | macOS | Custom Homebrew installation prefix |
| `DEVEX_WINGET_SOURCE` | Windows | Custom winget source repository |

## Shell Completion

DevEx provides comprehensive shell completion for all major shells, making command-line usage faster and more intuitive.

### devex completion

Generate shell completion scripts for your shell.

#### Usage

```bash
devex completion <shell> [flags]
```

#### Supported Shells

- **bash** - Bash shell completion
- **zsh** - Zsh shell completion  
- **fish** - Fish shell completion
- **powershell** - PowerShell completion (Windows)

#### Options

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--no-descriptions` | `bool` | `false` | Disable completion descriptions |

### Installation Instructions

<Tabs items={['Bash', 'Zsh', 'Fish', 'PowerShell']}>
  <Tab value="Bash">
    ```bash
    # Install bash completion
    devex completion bash > /etc/bash_completion.d/devex
    
    # Or for user-only installation
    devex completion bash > ~/.local/share/bash-completion/completions/devex
    
    # Reload your shell or run:
    source ~/.bashrc
    
    # Test completion
    devex <TAB><TAB>
    ```
  </Tab>
  
  <Tab value="Zsh">
    ```bash
    # Install zsh completion
    devex completion zsh > "${fpath[1]}/_devex"
    
    # Or add to your .zshrc:
    echo 'eval "$(devex completion zsh)"' >> ~/.zshrc
    
    # Reload your shell
    source ~/.zshrc
    
    # Test completion
    devex <TAB>
    ```
  </Tab>
  
  <Tab value="Fish">
    ```bash
    # Install fish completion
    devex completion fish > ~/.config/fish/completions/devex.fish
    
    # Reload fish configuration
    fish -c "source ~/.config/fish/config.fish"
    
    # Test completion
    devex <TAB>
    ```
  </Tab>
  
  <Tab value="PowerShell">
    ```powershell
    # Install PowerShell completion
    devex completion powershell | Out-String | Invoke-Expression
    
    # Add to PowerShell profile for persistence
    devex completion powershell >> $PROFILE
    
    # Test completion
    devex <TAB>
    ```
  </Tab>
</Tabs>

<Steps>

  </Step>
  
  <Step>
    **Flag Completion**
    ```bash
    devex install --<TAB>
    ```

  </Step>

    **Dynamic Content Completion**
    ```bash
    devex add <TAB>
    # Shows available applications: docker, git, node, python, vscode

    # Shows available templates: backend, frontend, full-stack

    # Shows configuration files: applications, environment, system, desktop
    ```

  </Step>
</Steps>

## Help System

DevEx provides comprehensive help at multiple levels.

### Command Help

```bash

devex --help
devex help

# Command-specific help
devex install --help
devex config --help

# Subcommand help
devex config edit --help
devex template apply --help
```

### Contextual Help

DevEx provides contextual help based on your current environment and configuration:

```bash
# Smart help suggestions
devex help
# Shows relevant commands based on your setup status

# Context-aware tips
devex status
# May suggest: "Run 'devex recovery check' to diagnose issues"

# Command suggestions
devex
# If no subcommand: "Try 'devex setup' for first-time configuration"
```

## Configuration Priority

<Steps>
  <Step>
    DevEx follows a clear priority order when determining configuration values:
    
    ### Command Line Flags (Highest Priority)
    ```bash
    devex install --config-dir ./custom
    # Always uses ./custom regardless of environment variables
    ```
  </Step>
  
  <Step>
    ### Environment Variables
    ```bash
    export DEVEX_CONFIG_DIR=./team-config
    devex install  # Uses ./team-config
    ```
  </Step>
  
  <Step>
    ### Configuration Files
    ```yaml
    # ~/.devex/config.yaml
    defaults:
      verbose: true
      backup: true
    ```
  </Step>
  
  <Step>
    ### Built-in Defaults (Lowest Priority)
    ```
    Default values hardcoded in DevEx
    ```
  </Step>
</Steps>

## Best Practices

<Callout type="tip">
**Global Flag Best Practices:**

1. **Use dry-run for safety** - Always preview major changes
2. **Enable verbose mode for debugging** - Get detailed information when troubleshooting
3. **Set up shell completion** - Improves command-line efficiency
4. **Use environment variables for consistency** - Set team-wide defaults
5. **Leverage help system** - Use `--help` for unfamiliar commands
</Callout>

### Common Patterns

```bash
# Safe operation pattern
devex install --dry-run    # Preview
devex install --verbose    # Execute with details

# Automation pattern
export DEVEX_AUTO_YES=true
export DEVEX_NO_COLOR=true
devex install --quiet

# Debugging pattern
export DEVEX_DEBUG=true
devex config validate --verbose

# Team consistency pattern
export DEVEX_CONFIG_DIR=/shared/team-configs
devex template apply backend-team
```
