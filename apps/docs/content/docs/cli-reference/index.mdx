# DevEx CLI Reference

The DevEx CLI provides a comprehensive suite of commands for managing your development environment. This reference covers all available commands, their options, and usage patterns.

<Callout type="info">
DevEx follows the Unix philosophy of composable tools. Each command has a specific purpose and can be combined with others for powerful workflows.
</Callout>

## Basic Usage

### Getting Started

```bash
# Start with the interactive setup
devex setup

# Or install everything with defaults
devex install

# Check what's installed
devex status
```

### Common Workflows

<Steps>
  <Step>
    **Initial Setup**: Use `devex setup` for first-time configuration
  </Step>
  <Step>
    **Add Tools**: Use `devex add <application>` to install additional software
  </Step>
  <Step>
    **Configure**: Use `devex config` to customize your environment
  </Step>
  <Step>
    **Maintain**: Use `devex status` and `devex recovery` for ongoing maintenance
  </Step>
</Steps>

## Global Flags

All DevEx commands support these global flags:

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--verbose` | `-v` | Enable verbose output | `false` |
| `--dry-run` | `-n` | Show what would be done without executing | `false` |
| `--config-dir` | | Override default config directory | `~/.local/share/devex/config` |
| `--help` | `-h` | Show command help | |

### Examples

```bash
# Verbose installation
devex install --verbose

# Dry-run to see what would be installed
devex add docker --dry-run

# Use custom config directory
devex install --config-dir ./custom-configs
```

## Environment Variables

DevEx recognizes these environment variables:

| Variable | Description | Default |
|----------|-------------|---------|
| `DEVEX_DEBUG` | Enable debug logging | `false` |
| `DEVEX_CONFIG_DIR` | Override config directory | `~/.local/share/devex/config` |
| `DEVEX_NO_COLOR` | Disable colored output | `false` |
| `DEVEX_CACHE_DIR` | Override cache directory | `~/.cache/devex` |

## Exit Codes

DevEx commands return standard exit codes:

- **0**: Success
- **1**: General error
- **2**: Invalid arguments or options
- **3**: Configuration error
- **4**: Installation error
- **5**: Platform not supported

## Performance Tips

<Callout type="tip">
- Use `--dry-run` to preview changes before execution
- Enable `--verbose` for detailed operation logging
- Use specific commands instead of running full `install` for single applications
- Set `DEVEX_CACHE_DIR` to a fast storage location for better performance
</Callout>

## Security Considerations

DevEx includes several security features:

- **Input validation**: All user inputs are validated and sanitized
- **Privilege escalation**: Only prompts for sudo when necessary
- **GPG verification**: Package signatures are verified when available
- **Audit trail**: All operations are logged for review

<Callout type="warn">
DevEx may require administrator privileges for system-level installations. Always review `--dry-run` output before executing privileged operations.
</Callout>

## Getting Help

For detailed help on any command:

```bash
# Command-specific help
devex <command> --help

# Interactive help system
devex help

# Version information
devex --version
```
