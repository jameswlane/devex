## Package Manager Support

### Windows Package Manager (WinGet)

<Callout type="success">
**Primary Package Manager** - Microsoft's official package manager
</Callout>

#### Planned Features
```bash
# DevEx will automatically use WinGet for installations
devex install git                    # Uses winget install Git.Git
devex install vscode                 # Uses winget install Microsoft.VisualStudioCode
devex install nodejs                 # Uses winget install OpenJS.NodeJS
```

#### Configuration Example
```yaml
# Planned Windows configuration format
applications:
  development:
    - name: visual-studio-code
      description: 'Code editor by Microsoft'
      windows:
        install_method: 'winget'
        install_command: 'Microsoft.VisualStudioCode'
        uninstall_command: 'Microsoft.VisualStudioCode'
        official_support: true
```

#### WinGet Features
- **Official Microsoft support**
- **Automatic updates** via Microsoft Store
- **Secure package verification**
- **Integration with Windows Update**

### Chocolatey

<Callout type="info">
**Community Package Manager** - Large ecosystem with extensive package availability
</Callout>

#### Planned Integration
```bash
# Alternative package manager with more packages
devex install --method=chocolatey firefox
devex install --method=chocolatey discord
devex install --method=chocolatey obs-studio
```

#### Chocolatey Benefits
- **Largest Windows package repository**
- **Community-maintained packages**
- **PowerShell-based installation**
- **Enterprise features available**

### Scoop

<Callout type="info">
**Developer-Focused** - Command-line installer for Windows
</Callout>

#### Planned Features
```bash
# Developer tools and CLI utilities
devex install --method=scoop git
devex install --method=scoop python
devex install --method=scoop nodejs
```

#### Scoop Advantages
- **Portable installations** (no admin required)
- **Developer-focused packages**
- **JSON manifest system**
- **Easy uninstallation**

## Installer Priority System (Planned)

DevEx will automatically select the best installer based on availability and package type:

```
1. winget      (Microsoft's official package manager)
2. chocolatey  (Community package manager)
3. scoop       (Developer-focused installer)
4. mise        (Programming language manager)
5. download    (Direct downloads and manual installation)
```

### Package Manager Selection Logic
```yaml
# Planned selection algorithm
selection_criteria:
  - official_support: prefer winget
  - package_availability: check all managers
  - installation_method: prefer non-admin when possible
  - update_mechanism: prefer automatic updates
```

## Windows Subsystem for Linux (WSL2) Integration

### Planned WSL2 Features
- **Automatic WSL2 setup** if not present
- **Linux distribution management** (Ubuntu, Debian, etc.)
- **Cross-platform configuration** sharing
- **Docker Desktop integration** via WSL2 backend

### Configuration Example
```yaml
# Planned WSL2 integration
wsl:
  enabled: true

  memory_limit: '8GB'
```

## Current Alternatives

While waiting for native Windows support, consider these alternatives:

### WSL2 + Linux DevEx
```bash
# Install WSL2 with Ubuntu
wsl --install -d Ubuntu-22.04

# Run DevEx inside WSL2 (review security guide first)
curl -fsSL https://devex.sh/install | bash
```

### Manual Installation
- **WinGet**: Use Microsoft's package manager directly
- **Chocolatey**: Community package manager
- **Scoop**: Developer-focused installations

### Docker Development Environment
```dockerfile
# Use DevEx in a container
FROM ubuntu:22.04
# Install DevEx with security considerations
RUN curl -fsSL https://devex.sh/install | bash
```

## Frequently Asked Questions

### When will Windows support be available?
**Target**: Version 2.0, estimated Q4 2025. Follow our [GitHub releases](https://github.com/jameswlane/devex/releases) for updates.

### Will it work on Windows 10?
**Yes**: Windows 10 version 1809+ will be supported with the same features as Windows 11.

### What about Windows Server?
**Under Review**: Server support is being evaluated based on community interest.

### Can I use DevEx with WSL2 now?
**Yes**: You can install DevEx inside WSL2 distributions and use full Linux functionality.

### Will there be a GUI installer?
**Future**: A graphical installer is planned for later releases, focusing on CLI implementation first.

See our [roadmap](https://github.com/jameswlane/devex/blob/main/ROADMAP.md) for the latest development status and timeline updates.
