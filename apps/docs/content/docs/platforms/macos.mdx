## Package Managers

### Homebrew (Primary)

<Callout type="info">
Homebrew is the primary package manager for macOS, providing the largest ecosystem of macOS-compatible software.
</Callout>

#### Installation & Setup
```bash
# DevEx automatically installs and configures Homebrew
devex install                    # Sets up Homebrew if needed

# Manual Homebrew verification
brew --version                   # Check installation
brew doctor                      # Diagnose issues
```

#### Package Types

**Formulae** (Command-line tools):
```bash
# Examples of formulae installations
devex install git               # Git version control
devex install nodejs            # Node.js runtime
devex install python            # Python interpreter
devex install docker           # Docker containerization
```

**Casks** (GUI Applications):
```bash
# Examples of cask installations  
devex install visual-studio-code    # VS Code editor
devex install google-chrome         # Chrome browser
devex install discord              # Discord client
devex install figma               # Figma design tool
```

#### Homebrew Configuration
```yaml
# Example DevEx configuration for macOS apps
applications:
  development:
    - name: visual-studio-code
      description: 'Code editor by Microsoft'
      macos:
        install_method: 'brew'
        install_command: '--cask visual-studio-code'
        uninstall_command: '--cask visual-studio-code'
        official_support: true
```

### Mac App Store (mas-cli)

<Callout type="warning">
**Limited Support** - Requires Apple ID authentication and app availability
</Callout>

#### Setup
```bash
# Install Mac App Store CLI
brew install mas

# Sign in to App Store (required)
mas signin your-apple-id@example.com
```

#### Usage
```bash
# Search for apps
mas search "Xcode"

# Install apps by ID
mas install 497799835            # Xcode
mas install 1295203466           # Microsoft Remote Desktop

# List installed apps
mas list
```

#### Configuration Example
```yaml
# Mac App Store configuration
applications:
  development:
    - name: xcode
      description: 'Apple development environment'
      macos:
        install_method: 'mas'
        install_command: '497799835'  # App Store ID
        official_support: true
```

## Platform-Specific Configuration

### Architecture Handling

DevEx automatically handles Apple Silicon vs Intel differences:

```yaml
# Cross-architecture configuration
applications:
  development:
    - name: docker
      description: 'Container platform'
      macos:
        install_method: 'brew'
        install_command: '--cask docker'
        platform_requirements:
          - architecture: 'arm64'    # Apple Silicon
            notes: 'Native Apple Silicon support'
          - architecture: 'x86_64'   # Intel  
            notes: 'Full Intel support'
```

### Rosetta 2 Integration

For Apple Silicon Macs running Intel-only software:

```bash
# DevEx automatically detects when Rosetta 2 is needed
devex install intel-only-app    # Prompts for Rosetta 2 if needed

# Manual Rosetta 2 installation
sudo softwareupdate --install-rosetta
```

## Troubleshooting

### Common Issues

#### Homebrew Issues
```bash
# Fix common Homebrew problems
brew doctor                      # Diagnose issues
brew update                      # Update Homebrew
brew upgrade                     # Upgrade packages

# Clear cache
brew cleanup                     # Remove old versions
rm -rf $(brew --cache)          # Clear download cache

# Reinstall Homebrew (if corrupted)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/uninstall.sh)"
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

#### Permission Issues
```bash
# Fix Homebrew permissions
sudo chown -R $(whoami) $(brew --prefix)/*

# Fix DevEx permissions  
sudo chown -R $(whoami) ~/.devex/
chmod -R 755 ~/.devex/
```

#### Path Issues
```bash
# Verify shell configuration
echo $PATH                       # Check current PATH
which brew                       # Should show Homebrew location

# Reload shell configuration
source ~/.zshrc                  # Zsh
source ~/.bash_profile          # Bash
```

### Architecture-Specific Issues

#### Apple Silicon
```bash
# Check if running under Rosetta
arch                            # Should show 'arm64'
uname -m                        # Should show 'arm64'

# Force native Apple Silicon
arch -arm64 brew install package

# Force Intel emulation (if needed)
arch -x86_64 brew install package
```

#### Intel Macs
```bash
# Verify Intel architecture
arch                            # Should show 'i386'
uname -m                        # Should show 'x86_64'

# Standard installation
brew install package
```

### Application-Specific Issues

#### Code Signing Issues
```bash
# Remove quarantine attribute
sudo xattr -r -d com.apple.quarantine /Applications/AppName.app

# Re-sign application (if needed)
sudo codesign --force --deep --sign - /Applications/AppName.app
```

#### Cask Issues
```bash
# Force reinstall cask
brew reinstall --cask app-name

# Clear cask cache
brew cleanup --prune=all
```

## Migration & Backup

### Migrating Between Macs
```bash
# Export DevEx configuration
devex config export --output ~/devex-config.yaml

# Import on new Mac
devex config import ~/devex-config.yaml
devex install                    # Install all configured apps
```

### Homebrew Migration
```bash
# Export Homebrew packages
brew bundle dump                 # Creates Brewfile

# Import on new Mac  
brew bundle install              # Installs from Brewfile
```

### Time Machine Integration
DevEx configurations are automatically included in Time Machine backups:
- `~/.devex/` - DevEx configuration
- `~/.zshrc` - Shell configuration  
- `~/Brewfile` - Homebrew packages (if created)

## Best Practices

### Security
1. **Keep Gatekeeper enabled** for application security
2. **Verify app sources** before installation
3. **Use official repositories** (Homebrew, Mac App Store)
4. **Regular updates** via DevEx and Homebrew

### Performance  
1. **Use Apple Silicon native apps** when available
2. **Minimize Rosetta 2 usage** on Apple Silicon
3. **Regular cleanup** of caches and old versions
4. **Monitor disk space** usage

### Organization
1. **Use consistent naming** for custom configurations
2. **Group applications** by category or project
3. **Document custom setups** for team sharing
4. **Version control** DevEx configurations
