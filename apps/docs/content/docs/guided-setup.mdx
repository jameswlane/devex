# Guided Setup

This guide walks you through setting up DevEx and configuring your development environment step by step.

## Prerequisites

Before starting, ensure you have:

- A supported operating system (Linux, macOS, or Windows)
- Internet connection for downloading packages
- Administrative privileges for system-level installations

### Supported Platforms

| Platform | Versions | Package Managers |
|----------|----------|------------------|
| **Ubuntu/Debian** | 20.04+ | APT, Flatpak, Snap |
| **Fedora/RHEL** | 35+ | DNF, Flatpak |
| **Arch Linux** | Rolling | Pacman, AUR |
| **macOS** | 11+ | Homebrew, Mac App Store |
| **Windows** | 10/11 | winget, Chocolatey, Scoop |

## Step 1: Install DevEx

Choose your preferred installation method:

### Quick Install (Recommended)

```bash
wget -qO- https://devex.sh/install | bash
```

This script will:
- Detect your operating system
- Download the appropriate DevEx binary
- Install it to `/usr/local/bin/devex`
- Set up basic configuration

### Manual Installation

#### Linux/macOS
```bash
# Download latest release
curl -L https://github.com/jameswlane/devex/releases/latest/download/devex-linux-amd64 -o devex

# Make executable and move to PATH
chmod +x devex
sudo mv devex /usr/local/bin/

# Verify installation
devex --help
```

#### Windows
```powershell
# Download from GitHub releases
Invoke-WebRequest -Uri "https://github.com/jameswlane/devex/releases/latest/download/devex-windows-amd64.exe" -OutFile "devex.exe"

# Move to PATH directory
Move-Item devex.exe C:\Windows\System32\devex.exe

# Verify installation
devex --help
```

## Step 2: Initial Configuration

DevEx works with sensible defaults, but you can customize it for your needs.

### View Default Configuration

```bash
# See what would be installed
devex install --dry-run
```

This shows all applications that would be installed across categories:
- Development tools (Git, VS Code, Docker)
- Programming languages (Node.js, Python, Go)
- System utilities (curl, wget, htop)
- Optional applications (configurable)

### Configuration File Locations

DevEx uses a two-tier configuration system:

```
~/.local/share/devex/config/    # Default configurations
├── applications.yaml           # Application definitions
├── environment.yaml           # Languages, fonts, shell
├── desktop.yaml              # Desktop environment settings
└── system.yaml               # Git, SSH, terminal settings

~/.devex/                      # User overrides (optional)
├── applications.yaml          # Override app selections
├── environment.yaml          # Override environment settings
├── desktop.yaml             # Override desktop settings
└── system.yaml              # Override system settings
```

## Step 3: Customize Your Setup

### Create User Configuration Directory

```bash
mkdir -p ~/.devex
```

### Customize Application Selection

Create `~/.devex/applications.yaml` to override defaults:

```yaml
categories:
  # Enable development tools
  development:
    git: true
    vscode: true
    docker: true
    postman: false  # Disable if not needed
    
  # Select programming languages
  languages:
    nodejs: true
    python: true
    go: true
    rust: false     # Disable if not needed
    
  # Choose databases
  databases:
    postgresql: true
    mysql: false
    redis: true
    
  # System utilities
  system:
    htop: true
    curl: true
    wget: true
    jq: true
    
  # Optional applications
  optional:
    slack: true
    discord: false
    spotify: true
    zoom: false
```

### Customize Environment Settings

Create `~/.devex/environment.yaml`:

```yaml
# Programming language versions
languages:
  nodejs:
    version: "20"
    packages:
      - "yarn"
      - "pnpm"
      - "@vue/cli"
      
  python:
    version: "3.11"
    packages:
      - "pip"
      - "pipenv"
      - "black"
      
  go:
    version: "1.21"

# Font preferences
fonts:
  - name: "JetBrains Mono"
    type: "monospace"
  - name: "Fira Code"
    type: "monospace"

# Shell configuration
shell:
  default: "zsh"
  oh_my_zsh: true
  plugins:
    - "git"
    - "docker"
    - "kubectl"
```

### Customize System Settings

Create `~/.devex/system.yaml`:

```yaml
# Git configuration
git:
  name: "Your Name"
  email: "your.email@example.com"
  default_branch: "main"
  
# SSH configuration
ssh:
  generate_key: true
  key_type: "ed25519"
  
# Terminal preferences
terminal:
  font: "JetBrains Mono"
  font_size: 12
  theme: "dark"
```

## Step 4: Run Installation

### Preview Changes

Always preview what will be installed:

```bash
devex install --dry-run
```

Review the output to ensure it matches your expectations.

### Install Development Environment

```bash
# Install all configured applications
devex install

# Or install specific categories
devex install development languages
```

The installation process will:
1. Detect your platform and available package managers
2. Install applications using the best available method
3. Configure programming language environments
4. Set up development tools and utilities

### Configure System Settings

```bash
# Configure Git, SSH, desktop environment
devex system
```

This will:
- Set up Git with your name and email
- Generate SSH keys if requested
- Configure desktop environment preferences
- Set up shell and terminal preferences

## Step 5: Verify Installation

### Check Installed Applications

```bash
# List all installed applications
devex uninstall

# Verify specific tools
git --version
node --version
docker --version
code --version
```

### Test Development Environment

```bash
# Create a test project
mkdir test-project && cd test-project

# Initialize Git repository
git init

# Create a simple Node.js project
npm init -y
echo 'console.log("DevEx setup successful!");' > index.js
node index.js
```

## Step 6: Team Setup (Optional)

### Share Configuration with Team

```bash
# Create team configuration repository
mkdir devex-team-config
cd devex-team-config

# Copy your configuration
cp -r ~/.devex/* .

# Initialize Git and push to remote
git init
git add .
git commit -m "Initial team DevEx configuration"
git remote add origin https://github.com/yourorg/devex-config.git
git push -u origin main
```

### Team Members Setup

Team members can use your configuration:

```bash
# Clone team configuration
git clone https://github.com/yourorg/devex-config.git ~/.devex

# Install team development stack
devex install
devex system
```

## Common Workflows

### Daily Development

```bash
# Update all applications
devex install

# Check for new configuration changes
git -C ~/.devex pull
```

### New Project Setup

```bash
# Install project-specific tools
devex install development databases

# Configure project environment
devex system
```

### CI/CD Integration

```bash
# Install minimal environment in CI
devex install --dry-run development | grep -q "docker" && devex install development
```

## Platform-Specific Notes

### Linux

- DevEx automatically detects your distribution
- Requires `sudo` privileges for system package installation
- Flatpak and Snap are used for applications not in system repositories

### macOS

- Homebrew is installed automatically if not present
- Xcode Command Line Tools may be required for some installations
- Mac App Store applications require signing in to the App Store

### Windows

- Windows Package Manager (winget) is preferred
- Some applications may require administrator privileges
- Windows Subsystem for Linux (WSL) is recommended for development
