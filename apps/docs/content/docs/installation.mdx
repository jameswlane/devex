# Installation Guide

DevEx provides multiple installation methods to suit different needs and preferences.

## Quick Install (Recommended)

The fastest way to get started with DevEx is using our one-line installer:

```bash
wget -qO- https://devex.sh/install | bash
```

This installer will:
1. **Download the latest DevEx release** for your platform
2. **Verify checksums** for security
3. **Install to** `~/.local/share/devex/`
4. **Update your PATH** in `~/.bashrc`
5. **Run the setup** with default configurations

### What Gets Installed

The quick installer automatically installs all applications marked with `default: true` in the configuration files:

- **Development tools**: Git, GitHub CLI, Neovim, fzf, ripgrep
- **Programming languages**: Node.js LTS (via mise)
- **System utilities**: curl, htop, tree
- **Desktop themes**: Tokyo Night theme for GNOME
- **Shell configuration**: Zsh with Oh My Zsh (if selected)

## Manual Installation

For more control over the installation process:

### 1. Download the Binary

Visit the [GitHub releases page](https://github.com/jameswlane/devex/releases) and download the appropriate binary for your system:

```bash
# Example for Linux x86_64
wget https://github.com/jameswlane/devex/releases/latest/download/devex_linux_amd64.tar.gz

# Extract
tar -xzf devex_linux_amd64.tar.gz

# Make executable
chmod +x devex

# Move to PATH
sudo mv devex /usr/local/bin/
```

### 2. Initialize Configuration

```bash
# Create configuration directory
mkdir -p ~/.local/share/devex/config

# Download default configurations
devex config init
```

### 3. Run Setup

```bash
# Install with dry-run to preview changes
devex install --dry-run

# Install default applications
sudo devex install

# Or install specific categories
sudo devex install --category development
```

## Platform-Specific Instructions

### Linux (Ubuntu/Debian)

```bash
# Prerequisites
sudo apt update
sudo apt install -y wget curl

# Install DevEx
wget -qO- https://devex.sh/install | bash
```

### Linux (Fedora/RHEL)

```bash
# Prerequisites
sudo dnf install -y wget curl

# Install DevEx
wget -qO- https://devex.sh/install | bash
```

### Linux (Arch)

```bash
# Prerequisites
sudo pacman -S wget curl

# Install DevEx
wget -qO- https://devex.sh/install | bash
```

### macOS

```bash
# Using Homebrew (recommended)
brew install jameswlane/tap/devex

# Or using the installer
curl -fsSL https://devex.sh/install | bash
```

### Windows

```powershell
# Using winget
winget install jameswlane.devex

# Or download from releases page
# https://github.com/jameswlane/devex/releases
```

## Docker Installation

For containerized environments:

```bash
# Run DevEx in a container
docker run --rm -it \
  -v $HOME/.devex:/root/.devex \
  -v $HOME/.local/share/devex:/root/.local/share/devex \
  jameswlane/devex:latest

# Or build from source
git clone https://github.com/jameswlane/devex.git
cd devex
docker build -t devex .
```

## CI/CD Integration

### GitHub Actions

```yaml
name: Setup Development Environment
on: [push, pull_request]

jobs:
  setup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Install DevEx
        run: |
          wget -qO- https://devex.sh/install | bash
          echo "$HOME/.local/share/devex/bin" >> $GITHUB_PATH
      
      - name: Setup environment
        run: devex install --category development
```

### GitLab CI

```yaml
setup_environment:
  stage: setup
  image: ubuntu:latest
  before_script:
    - apt-get update && apt-get install -y wget curl sudo
  script:
    - wget -qO- https://devex.sh/install | bash
    - export PATH="$PATH:$HOME/.local/share/devex/bin"
    - devex install --dry-run
```

## Verification

After installation, verify DevEx is working correctly:

```bash
# Check version
devex --version

# Validate configuration
devex config validate

# List available installers
devex install --list

# Check system compatibility
devex system info
```

## Uninstallation

To completely remove DevEx:

```bash
# Remove installed applications (if desired)
devex uninstall --all

# Remove DevEx binary and data
rm -rf ~/.local/share/devex
rm -rf ~/.devex

# Remove from PATH (edit ~/.bashrc)
# Remove the line: export PATH="$PATH:$HOME/.local/share/devex/bin"
```

## Troubleshooting

### Permission Issues

If you encounter permission errors:

```bash
# Ensure proper ownership
sudo chown -R $USER:$USER ~/.local/share/devex
sudo chown -R $USER:$USER ~/.devex

# Fix binary permissions
chmod +x ~/.local/share/devex/bin/devex
```

### Network Issues

If downloads fail:

```bash
# Use curl instead of wget
curl -fsSL https://devex.sh/install | bash

# Or set proxy
export https_proxy=http://your-proxy:port
wget -qO- https://devex.sh/install | bash
```

### Configuration Issues

If configuration loading fails:

```bash
# Reset to defaults
devex config reset

# Validate configuration
devex config validate

# Debug mode
devex --verbose install --dry-run
```
